# Workfront Admin Console登录页面

准备将您的组织载入Adobe Admin Console [迁移检查列表](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/admin-in-admin-console/prep-for-admin-console.html?lang=zh-Hans)：

**迁移检查列表**

* 确定要在其中添加Workfront的所需Adobe Admin Console。

  如果您已经拥有其他Adobe产品，请确认您是否愿意将Adobe Workfront添加到当前的Adobe Admin Console。 如果您需要更多信息，请使用[此链接](https://helpx.adobe.com/cn/enterprise/using/admin-console.html)了解管理Adobe产品的最佳实践。

* 请参阅[以下链接](https://helpx.adobe.com/cn/enterprise/using/deployment-planning.html)以获取有关部署的帮助。
* 如果使用单点登录，您将需要将新的Adobe Admin Console连接到现有的SSO提供商。
* 确保删除重复的电子邮件，并且只使用有效的电子邮件地址。
* 如果您使用的是自定义API集成，请确认这些用户具有他们可以访问的有效电子邮件地址。

  Workfront建议将所有自定义集成更新为[利用OAuth2](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/configure-integrations/create-oauth-application.html?lang=zh-Hans)。

配置主机后，请参阅此链接以查看[如何在Adobe Admin Console中管理用户](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/add-users/create-manage-users/admin-console.html?lang=zh-Hans)。
